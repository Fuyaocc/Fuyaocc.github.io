java Guide基础篇

java语言特点

Java通过编译器编译成.class文件(字节码文件)不能直接被os识别运行，需要通过 JVM 从.class文件中读一行解释一行。

  • 编译和解释并存：从 字节码 变成 机器码 这一步JVM中有 解释器 和 JIT编译器 。
    • 解释器就负责将 解释成机器码执行。
    • 由于有些方法需要经常被调用，所以使用JIT编译器(属于运行时编译)。当 JIT 编译器完成第一次编译后，其会将字节码对应的机器码保存下来，下次可以直接使用。
  • 简单易学、面向对象（封装，继承，多态）、平台无关性（ Java 虚拟机实现平台无关性）、支持多线程、可靠性、安全性、支持网络编程并且很方便

Java的执行过程

左边为 前端编译 ，右边为 后端编译 。后端编译中一段代码的调用次数只有超过阈值才能被 JIT 编译并存入codeCache。

JVM的跨平台性

  • JVM是负责运行java字节码 的虚拟机，JVM有多个OS的版本，为了运行统一的字节码文件，实现跨平台。

static与final修饰符

  • final修饰的变量会变成常量，且必须 显示赋值
  • static修饰的变量会变成静态变量，static修饰的成员变量属于类且被所有类的实例共享，没有static的成员变量属于生成的对象。
  • 静态方法在访问本类的成员时，只允许访问静态成员（即静态成员变量和静态方法），不允许访问实例成员（即实例成员变量和实例方法），而实例方法不存在这个限制。
    • 原因 ：
    • 静态方法属于类，在类的加载时就会分配内存，而非静态成员属于实例，只有实例被创建后才存在。
    • 在类的非静态成员不存在的时候静态成员就已经存在了，此时调用在内存中还不存在的非静态成员，属于非法操作。

字符常量特点

  • char 在 Java 中占两个字节，是因为 Java 中使用 Unicode 字符，所有字符均以2个字节存储。

重载和重写的区别

区别点 重载方法 重写方法
发生范围 同一个类 子类
参数列表 必须修改 一定不能修改
返回类型 可修改子类方法返回值类型 应比父类方法返回值类型更小或相等
异常 可修改子类方法声明抛出的异常类 应比父类方法声明抛出的异常类更小或相等
访问修饰符 可修改 一定不能做更严格的限制(可以降低限制)
发生阶段 编译期 运行期

重载发生在编译期： 根据参数，返回值等不同，编译器通过用各个方法给出的参数匹配来挑选出相应的方法。 如果编译器找不到匹配的参数， 就会产生编译时错误， 因为根本不存在匹配， 或者没有一个比其他的更好(这个过程被称为重载解析(overloading resolution))。

方法的重写要遵循“两同两小一大”

  • “两同”即方法名相同、形参列表相同；
  • “两小”指的是子类方法返回值类型应比父类方法返回值类型更小或相等，子类方法声明抛出的异常类应比父类方法声明抛出的异常类更小或相等；
    • 注：若返回值类型是 引用类型 ，则可以返回该 引用类型的子类 。若返回类型是 void 和 基本数据类型 ，则返回值重写时不可修改。
  • “一大”指的是子类方法的访问权限应比父类方法的访问权限更大或相等。

基本类型和包装类型的区别

  • 包装类型不赋值为 ,基本数据类型都有初值
  • 包装类型可用于泛型，基本数据类型不可
  • 基本数据类型的存放
    变量 存放位置
    成员变量 Java 虚拟机 栈区的局部变量表
    非静态成员变量 Java虚拟机的堆区
    静态成员变量 方法区
  • 包装类型属于对象类型，几乎所有 对象实例都存在于堆中。
    • 因为 HotSpot 虚拟机引入了 JIT 优化之后，会对对象进行逃逸分析，如果发现某一个对象并没有逃逸到方法外部，那么就可能通过标量替换来实现栈上分配，而避免堆上分配内存
    • 逃逸：一个对象在方法中定义之后，它可能被外部方法所引用，作为参数传递到其它方法中，这叫做方法逃逸

包装类型的缓存机制

  • Byte,Short,Integer,Long 这 4 种包装类默认创建了 的缓存数据 Character 创建了 的缓存数据，Boolean 直接返回 True or False。Float,Double 并没有实现缓存机制
  • 如果超出对应范围仍然会去创建新的对象，缓存的范围区间的大小只是在性能和资源之间的权衡。
  • Integer缓存源码

整型包装类对象之间值的比较，全部使用 equals 方法比较

自动装箱与拆箱

  • 装箱 ：将基本类型用它们对应的引用类型包装起来，调用了包装类的valueOf()方法
  • 拆箱 ：将包装类型转换为基本数据类型，调用了 xxxValue()方法

浮点数运算的精度丢失

  • 原因：变量存储的位数有限，在遇到无限小数(比如 )时只能进行截断。
  • BigDecimal 可以实现对浮点数的运算，不会造成精度丢失。
    • 原理：十进制整数在转化成二进制数时不会有精度问题，那么把十进制 小数扩大N倍 让它在整数的维度上进行计算，并保留相应的精度信息。

超过 long 整型的数据

  • 使用 BigInteger ，BigInteger 内部使用 int[] 数组来存储任意大小的整形数据.但运算效率会较低。
