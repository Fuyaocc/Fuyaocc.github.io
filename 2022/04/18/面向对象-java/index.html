面向对象

面向对象方法三个基本特征: 封装 , 继承 , 多态

封装

访问控制符

private ：只能在当前类的内部被使用

protected ：可以被当前类的内部使用，也可以被不同包内的子类使用

public ：可以被所有外部类所使用

default ：之恩那个被相同包内的类访问

继承

  • 若一个类未显式指定这个类的直接父类，则这个类默认扩展 java.lang.Object类，Object类是所有类的基类。
  • 重写：子类重写父类的方法，两同两小一大 原则：方法名和形参列表相同，返回值类型比父类方法的相等或更小，子类方法的访问权限比父类方法的更大或相等。
  • 若父类中有private修饰的方法，子类访问不到所以即便有相同函数名的函数也不是重写
  • 系统会为子类从父类继承的所有实例变量分配内存。
  • 父类构造器总会在子类构造器之前执行，子类若没super调用父类构造器，也没this调用，系统在执行子类构造器之前会隐式调用父类的无参构造器。

super

  • 如果需要在子类方法中调用父类被重写的实例方法，则可使用 super 限定来调用父类被重写的实例。比如 super.function()
  • 子类有父类的同名变量，也可使用 super 调用
  • 子类可通过 super 调用父类的构造器

多态

Java 引用变量有两个类型，一个是编译时类型，一个是运行时类型。编译时类型由声明该变量时使用的类型决定，运行时类型由实际赋给该变量的对象决定，如果编译时类型和运行时类型不一致，就可能出现所谓的多态。

比如： father obj=new child(); obj对象的编译时类型是father，运行时类型是child类型。
obj可以调用父类的方法并且若子类重写父类的方法，则调用的方法呈现出子类方法的行为特征 _(即类似于父类方法变成了子类的方法)_，但obj对象不能调用子类的方法。

  • 引用变量的强制类型转换只能发生在具有继承关系的两个类之间。

动态绑定

  • 当调用对象方法的时候， 该方法和该对象的内存地址也就是运行时类型 绑定
  • 当调用对象属性时候，没有动态绑定，哪里声明哪里使用

instanceof

  • instanceof运算符可以判断引用类型强制类型转换是否成功。

  • 所有类都是引用类型

类三种成员(四种)

方法

  • 方法不能独立定义，必须在类内部定义
  • 方法要么属于该类本身，要么属于该类的一个对象
  • 执行方法必须使用类或者其对象来调用，不能独立执行
  • 重载：方法名相同，形参列表不相同

参数传递

只有 值传递 ：将实际参数的副本传入方法内
传入的是引用类型会将该引用类型的值即 堆内地址 复制一份给局部变量，也是值传递

成员变量和局部变量

成员变量初始化和内存中的机制

  • 创建一个类的对象时，若是第一次使用该类的时候，系统会先加载这个类，并且初始化，为其静态成员变量分配内存，并指定默认初始值，类不保存非静态成员变量。
  • 创建对象时，只会给非静态成员变量分配内存
  • 成员变量都保存在堆内存中

局部变量初始化和内存中的机制

  • 局部变量必须经过显式初始化
  • 局部变量保存在方法的栈内存中，无需系统垃圾回收，随着方法或代码块的运行结束而结束

构造器

  • 构造器在创建对象的时候执行初始化，无自定义的构造器是会有默认构造器

初始化块

  • 初始化块的修饰符只能是static
  • 初始化块的执行顺序和其在代码的的位置顺序有关
  • 初始化块总在构造器之前执行
  • 静态初始块总比普通初始块先执行
    初始化块在编制结束之后会被“还原”到每个构造器中，并且位于构造器所有代码的前面。

== 和 equal 方法

  • == 和 equal 方法：当两个变量为数值类型的时候且值相等时 == 返回true，当两个变量为引用类型的时候,只有当两个变量指向同一个对象时 == 才返回true， equal 方法与 == 没有区别。
  • JVM常量池：专门用于管理在编译时被确定并被保存在已编译的class文件中的一些数据 它包括了关于类、方法、接口中的常量，还包括字符串常量。
  • JVM常量池保证相同的字符串直接量只有 1个 ，不会产生多个副本

static关键字

  • static 修饰的成员不能访问没有 static 修饰的成员，即 静态成员不能直接访问非静态成员
  • static 修饰的成员表明它属于这个类本身，而不属于该类的单个实例，不使用 static 修饰的普通方法、成员变变量则属于该类的单个实例，而不属于该类。
  • static 修饰的方法和成员变量，既可通过类来调用，也可通过实例来调用 (不建议使用对象来调用静态成员) ,没有使用 static 修饰的成员变量和方法都必须使用对象来调用。
  • static 修饰的方法中不能使用 this 引用

final修饰符

final修饰的类、方法和变量不可改变

final成员变量

  • final修饰的成员变量必须有程序员显式地指定初始值
  • final修饰的类变量必须在静态初始化块、声明该变量时二者中选一个指定初始值
  • final修饰的实例变量把必须在非静态初始化块、声明该变量、构造器三者中的一个内指定初始值

final局部变量

  • 系统不会对局部变量初始化，需自己显式初始化。所以final 修饰局部变量时 既可以在定义时指定默认值，也可以不指定默认。

final修饰引用变量

  • final修饰引用变量时候，只保证这个引用类型变量所引用的地址不会改变，即引用同一个对象，但这个对象内的成员是可以发生改变的。

final修饰方法

  • final修饰的方法不可以被重写，但是可以被重载。

final修饰类

  • final修饰的类不能有子类。

abstract抽象类、抽象方法

  • 抽象类体现了 模板模式 这一个设计模式。
  • 有抽象方法的类是抽象类，抽象类不一定有抽象方法。
  • 抽象类可以有普通方法、抽象方法、成员变量、初始化块构造器等。
  • 抽象方法不能有方法体，抽象类不能被实例化。
  • 子类继承抽象类必须实现抽象类的所有抽象方法。
  • abstract 和 static 不能同时修饰某个方法，但可以同时修饰内部类。

interface接口

接口是特殊的抽象类

  • 接口里不包含构造器和初始化块定义。
  • 接口里可以包含成员变量(只能是静态常量)、方法(只能是抽象实例方法、静态方法、默认方法或私有方法)、静态内部类(包括内部接口、枚举)定义
  • 接口支持多继承

内部类

非静态内部类

  • 非静态内部类内不允许定义静态成员
  • 非静态内部类可以直接访问外部类的所有变量和方法
  • 外部类不能直接访问非静态内部类的成员变量和方法,外部类必须创建非静态内部类的实例访问其实例成员。
  • 外部类静态成员不能直接使用非静态内部类

外部类以外的地方创建非静态内部类实例必须使用外部类实例和new 来调用非静态内部类的构造器。

静态内部类

  • 静态内部类属于外部类本身
  • 静态内部类不能访问外部类的实例成员(非静态成员)，只能访问外部类的类成员(即静态成员)。
  • 外部类不能直接访问静态内部类成员，可以通过静态内部类的类名或者静态内部类的对象作为调用者来访问。

对象与垃圾回收

jvm垃圾回收由系统自动执行，有不确定性。

对象在内存中的状态

在实例化一个类的时候，类的对象保存在堆(heap)中，栈中保存对该对象的引用，指向对象在堆内中的地址。
对象在内存中的三种状态: 可达状态 , 可恢复状态 , 不可达状态

强制垃圾回收

  • 程序可以强制 (通知建议) 系统进行垃圾回收，但系统是否进行回收不确定。

finalize 方法

  • 在垃圾回收机制回收任何对象之前，总会先调用它的 finalize()方法，该方法可能使该对象重新
    复活(让一个引用变量重新引用该对象) ，从而导致垃圾回收机制取消回收。
  • finalize不一定会执行
  • 不需要程序员指定，系统自动调用

引用的类型

  • 硬引用：把对象直接赋值个一个引用变量。
  • 软引用：使用SoftReference，在内存够时不会被垃圾回收，内存不够时候可能会被回收
  • 弱引用：使用WeakRederence，系统垃圾回收机制运行时候，总会被回收。
  • 虚引用：类似于没引用，需和引用队列联合使用，作用是跟踪对象被垃圾回收的状态
